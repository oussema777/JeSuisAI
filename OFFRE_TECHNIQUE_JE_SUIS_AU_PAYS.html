1. Executive Summary & Scope

The objective of this engineering cycle is to pivot the current Je Suis Au Cameroun (JSAC) monolith into a Multi-Tenant Regionalized Platform (JSAP). The system must support N-countries through a single core engine, with strict data isolation, dynamic regionalization, and multi-domain mapping.

2. System Architecture Design

[HTTP REQUEST] -> [NEXT.JS MIDDLEWARE (Domain/Path Detection)]
       |
       V
[TENANT CONTEXT INJECTION (CountryID, Locale, Theme)]
       |
       +---> [API / DATABASE (RLS Filtered by CountryID)]
       +---> [UI LAYER (Themed via CSS Variables)]

3. Database Schema Evolution (Supabase)

A new countries metadata table is required to drive the entire regionalization engine.

Table New Column / Action Technical Note
countries (New Table) Stores: id, slug, iso_code, domain_alias, theme_config (JSON).
opportunites country_id (FK) Mandatory for mission isolation.
profiles country_id (FK) Users are tied to their primary regional registration.
actualites country_id (FK) News feeds filtered by region.

SQL: Security Hardening (RLS)

-- Example of Row Level Security for Multi-Tenancy
ALTER TABLE public.opportunites ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only see missions for their country" 
ON public.opportunites
FOR SELECT 
USING (country_id = (SELECT country_id FROM profiles WHERE id = auth.uid()));

4. Routing Strategy & Domain Mapping

We will implement a hybrid routing strategy using Next.js Middleware to handle both Path-based and Domain-based access.

Source Domain Detected Tenant Final Internal URL
jesuisaucameroun.com Cameroun /cm/fr/*
jesuisautunisie.com Tunisie /tn/fr/*
jesuisaupays.com/cm Cameroun /cm/fr/*

Middleware Implementation Logic

// middleware.ts (Pseudo-code)
export function middleware(req) {
  const hostname = req.headers.get('host');
  const tenant = await getTenantByDomain(hostname) || getTenantByPath(req.nextUrl.pathname);
  
  if (tenant) {
    return NextResponse.rewrite(new URL(`/${tenant.slug}${req.nextUrl.pathname}`, req.url));
  }
}

5. Dynamic Theme Engine (CSS-in-DB)

Branding will no longer be hardcoded. It will be injected as CSS variables at the Layout level.

/* The Layout will inject these based on the Tenant Config */
:root {
  --primary-color: ${tenant.theme.primary};
  --secondary-color: ${tenant.theme.secondary};
  --hero-image: url('${tenant.theme.hero}');
}

7. Risk Assessment & Mitigation

Risk: SEO Dilution
Mitigation: Strict 301 permanent redirects from old domains to new sub-paths to preserve "Link Juice".

Risk: Data Cross-Leakage
Mitigation: Database-level RLS policies that act as a hard firewall even if the frontend fails.
