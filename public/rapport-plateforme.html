6 Backend & Infrastructure

6.1 — Authentification & Sécurité

  • Authentification email/mot de passe via Supabase Auth
  • 4 rôles utilisateur : Superadmin, Admin, Annonceur, user
  • Protection middleware sur toutes les routes /admin et /superadmin
  • Sessions sécurisées via cookies (SSR compatible)
  • Headers de sécurité : CSP, HSTS, X-Frame-Options, Permissions-Policy
  • Row Level Security (RLS) sur toutes les tables Supabase
  • Isolation multi-tenant par annonceur_id
  • Rate limiting sur les API sensibles (email, invitation, notification)

Tous les correctifs critiques identifiés lors de l'audit ont été implémentés, testés et vérifiés. Le build passe sans erreur avec la vérification TypeScript activée.
Tâche Détails de la correction Statut
Activer la vérification TypeScript ~80 erreurs TypeScript corrigées dans 10 fichiers. Le flag ignoreBuildErrors est désormais à false. Le build de production compile sans erreur. Fait
Corriger la vulnérabilité XSS dans les emails escapeHtml() appliqué à toutes les données utilisateur dans sendRecapEmail.ts (recipientName, labels, valeurs). Audit des 6 templates email : les 5 autres étaient déjà protégés. Fait
Protection anti brute-force sur le login Nouvelle route API /api/auth/login servant de proxy serveur avec double rate-limiting : 5 tentatives / 15 min par IP et par email. Le client appelle désormais ce proxy au lieu de Supabase directement. Fait
Validation serveur pour les uploads Nouvelle route API /api/upload avec validation MIME type (whitelist), extension, taille (max 10 Mo) et bucket autorisé. Les 3 pages publiques (Postuler, Soumettre Profil, Soumettre Projet) utilisent désormais cette API. Fait
